
## Computer Architecture: The Big Picture

Computer architecture is the design and organization of computer systems. It encompasses how hardware and software(at binary level) interact to perform computations.

Key aspects of computer architecture include:
- **Instruction Set Architecture (ISA)**: The programmer-visible interface
- **Microarchitecture**: The implementation details of the ISA
- **System Architecture**: How components interconnect (memory, I/O, etc.)

Computer architecture sits between computer science (algorithms, software) and electrical engineering (circuits, physics). It determines what is possible in computing and influences performance, power, cost, and reliability.

Architectures are classified by their design philosophies, with two major paradigms dominating modern computing: CISC and RISC.

## What is an Instruction Set Architecture (ISA)?

An Instruction Set Architecture (ISA) is the interface between software and hardware in a computer system. It defines the set of instructions that a CPU can execute, including the operations, data types, registers, and memory addressing modes. The ISA acts as a contract: software written for a specific ISA can run on any hardware that implements that ISA, regardless of the underlying implementation details.

Think of the ISA as the "language" that the CPU understands. Just like human languages have vocabulary and grammar, an ISA has a defined set of instructions and rules for how they work.

For example, an opcode like '00000001' may represent the operation 'add' in one ISA, while the binary form of that instruction may be 00001110 in another ISA.For precision, note that this represents only a part of an instruction, not the complete instruction.

## What is CISC (Complex Instruction Set Computer)?

CISC stands for Complex Instruction Set Computer. It's a design philosophy for CPU architectures where the instruction set includes many complex instructions that can perform multiple operations in a single instruction. These instructions are designed to closely match high-level programming constructs, making it easier for compilers to generate efficient machine code.

Key characteristics of CISC:
- **Complex Instructions**: Instructions can perform multiple operations (e.g., memory access + computation + branching)
- **Variable-Length Instructions**: Instructions can be different sizes
- **Rich Addressing Modes**: Many ways to access memory and data
- **Microcode**: Complex instructions are often implemented using microcode (internal CPU programs)
- **Emphasis on Hardware Complexity**: More work done by hardware, less by software

CISC contrasts with RISC (Reduced Instruction Set Computer), which uses simpler, fixed-length instructions that execute quickly but may require more instructions to accomplish the same task.

## What is RISC (Reduced Instruction Set Computer)?

RISC is a design philosophy that emphasizes:
- a small, highly optimized set of instructions
- fixed instruction length
- simple addressing modes
- load/store architecture (separate memory access from ALU operations)
- fast, predictable instruction execution

CISC ISAs like x86 prioritize ease of programming and code density over simplicity of implementation.

## Introduction to x86

x86 is a family of backward-compatible instruction set architectures originally developed by Intel in the 1970s. The name "x86" comes from the fact that the first processors in this family ended with "86" (like 8086, 80286, 80386, etc.).

x86 is the most widely used ISA in personal computers and servers. It powers the vast majority of desktop and laptop computers, as well as many enterprise systems. Despite its age, x86 continues to evolve and remains dominant in the computing world.

## History and Evolution of x86

- **1978: Intel 8086** - The original 16-bit processor with 20-bit addressing (1 MB address space)
- **1982: Intel 80286** - Added protected mode for multitasking
- **1985: Intel 80386** - First 32-bit processor (80386), introduced flat memory model
- **1993: Intel Pentium** - Superscalar architecture, MMX instructions
- **1997: Intel Pentium II** - Slot 1 design, improved performance
- **1999: Intel Pentium III** - SSE instructions for multimedia
- **2000: Intel Pentium 4** - NetBurst architecture
- **2006: Intel Core 2** - Return to efficient design
- **2003: AMD Athlon 64** - First mainstream 64-bit x86 processor
- **2006: Intel Core 2 Duo** - Multi-core revolution
- **2011: Intel Sandy Bridge** - Integrated graphics, AVX instructions
- **2015: Intel Skylake** - Continued evolution
- **2017: Intel Coffee Lake** - Higher core counts
- **2020s: Intel 11th/12th/13th Gen** - Hybrid architecture with performance/efficiency cores

Throughout its history, x86 has maintained backward compatibility - software written for the 8086 can still run on modern x86 processors.

## x86 as a CISC ISA

x86 is an  example of a CISC architecture. Here's why:

### Complex Instructions
x86 includes instructions that perform multiple operations:
- `REP MOVSB`: Copies a block of memory (equivalent to a loop in software)
- `ENTER`/`LEAVE`: Set up/tear down stack frames for function calls
- `LOOP`: Decrements counter and branches if not zero
- String instructions (`MOVSB`, `CMPSB`, etc.) that operate on arrays

### Variable-Length Instructions
x86 instructions can be 1 to 15 bytes long, allowing for compact encoding of complex operations while still supporting simple ones.

### Rich Addressing Modes
x86 supports numerous ways to access memory:
- Immediate: `MOV EAX, 42`
- Register: `MOV EAX, EBX`
- Direct: `MOV EAX, [0x1000]`
- Register indirect: `MOV EAX, [EBX]`
- Base + displacement: `MOV EAX, [EBX + 8]`
- Base + index: `MOV EAX, [EBX + ECX]`
- Base + index + displacement: `MOV EAX, [EBX + ECX + 8]`
- Scaled index: `MOV EAX, [EBX + ECX*4]`

### Microcode Implementation
Many complex x86 instructions are implemented using microcode - small programs stored in the CPU that break down complex operations into simpler steps.

### Hardware Complexity
x86 CPUs include complex circuitry to decode and execute these varied instructions efficiently, including:
- Out-of-order execution
- Branch prediction
- Multiple execution units

## What is ARM?

ARM (originally Acorn RISC Machine, now Advanced RISC Machines) is a family of RISC ISAs designed for low power, high efficiency, and clean architectural scaling. ARM is widely used in mobile devices, embedded systems, and increasingly in laptops and servers.

Key ARM (RISC) traits:
- fixed-length instructions (mostly 32-bit in classical ARM or 16-bit/32-bit in Thumb)
- simple, regular instruction formats
- load/store architecture (only load/store instructions access memory; ALU ops only use registers)
- uniform registers (e.g., R0-R15 in ARM) and strong convention for register usage
- fewer addressing modes compared to x86

### ARM vs x86 (CISC vs RISC)

- Instruction complexity:
  - x86: many complex, specialized instructions; variable length
  - ARM: small set of simpler instructions; consistent length (mostly)

- Decoding:
  - x86: complex decoder pipeline to handle variable-length fields, prefixes, addressing modes
  - ARM: simpler decode hardware, fewer instruction formats

- Memory access model:
  - x86: allows memory operands in many arithmetic instructions (e.g., `ADD EAX, [EBX]`)
  - ARM: load/store model, operations on registers only (`LDR`, `STR` for memory)

### Explanation of the examples used in MAM

Task: Add the value at the memory address stored in a register to another register.
ISA Specific Simplification:
Add value at memory address stored in `EBX` to `EAX` (Registers of x86 Chip)
Add value at memory address stored in `R2` to `R0`(Registers of ARM chip)

- x86 (CISC, direct memory operand allowed):
  - `ADD EAX, [EBX]`        ; EAX =  value in EAX + the value at the memory address stored in EBX register

- ARM (RISC, load/store requires explicit load):
  - `LDR R1, [R2]`          ; R1 = the value at the memory address stored in R2 Register
  - `ADD R0, R0, R1`        ; R0 = R0 + R1

Why this matters:
- x86 can do memory + ALU in one instruction, giving compact code but greater decoder/hardware complexity.
- ARM requires separate memory access and ALU operation, giving simpler pipeline and predictable timing.

- Register set:
  - x86: fewer general-purpose registers (e.g., RAX, RBX, RCX...).
  - ARM: large uniform register file (e.g., 16 or 31 registers in modern 64-bit AArch64 )

AArch64 = ARM’s 64-bit Instruction Set Architecture

- Code density and power:
  - x86: historically lower code density due to richer instruction forms, more hardware overhead
  - ARM: designed for code density and energy efficiency, especially in mobile applications

- Backward compatibility:
  - x86: strong commitment (old binaries run), leading to ISA complexity growth
  - ARM: explicit architectural versions with some compatibility but less “full backward-compatibility” legacy baggage

### ARM's place in the ecosystem

- Dominates smartphones and many IoT devices (e.g., Cortex-A, Cortex-M lines)
- Used in Apple Silicon (A-series and M-series), Qualcomm, Samsung, etc.
- Gaining server adoption (AWS Graviton).

### Why compare ARM and x86?

Understanding both shows the trade-offs in CPU architecture:
- x86 invests in complex ISA and huge compatibility; this can produce high single-thread peak throughput at high power
- ARM invests in a clean, low-power RISC pipeline; this can produce high efficiency, especially in parallel and low-power contexts

Both families continue improving with techniques borrowed from each other (e.g., ARM adds complex instruction hints, x86 adds power gating and improved branch prediction).

## Key Features of x86 Architecture
This documentation will cover assembly for both x86 and ARM ISAs. First we need a better understanding of x86 CPUs because they have some complex instructions that use registers implicitly. For x86, we start from 16-bit (real mode) and concurrently explain ARM + x86 at the 64-bit level.
### Registers
x86-64 (64-bit x86) has 16 general-purpose registers:
- `RAX`, `RBX`, `RCX`, `RDX` - General purpose (historically had special roles)
- `RBP`, `RSP` - Base pointer and stack pointer
- `RSI`, `RDI` - Source and destination for string operations
- `R8` through `R15` - Additional general-purpose registers (64-bit extension)

Each register can be accessed in different sizes:
- 64-bit: `RAX`
- 32-bit: `EAX`
- 16-bit: `AX`
- 8-bit: `AL`, `AH`

### Data Types
x86 supports various data sizes:
- Bytes (8 bits)
- Words (16 bits)
- Doublewords (32 bits)
- Quadwords (64 bits)
- And larger with SIMD extensions

### SIMD Extensions
x86 has been extended with Single Instruction, Multiple Data (SIMD) capabilities:
- MMX (1997): 64-bit SIMD for multimedia
- SSE (1999): 128-bit SIMD, streaming operations
- AVX (2008): 256-bit SIMD
- AVX-512 (2013): 512-bit SIMD for high-performance computing

### Floating Point
x86 includes x87 floating-point instructions for single, double, and extended precision floating-point arithmetic.

## Why x86 Matters for Assembly Programming

x86 is the primary target for assembly language education because:
1. **Ubiquity**: Most computers run x86, so assembly skills are practically useful
2. **Complexity**: x86's CISC nature provides rich examples of architectural concepts
3. **Evolution**: x86 shows how architectures evolve while maintaining compatibility
4. **Performance**: Understanding x86 assembly helps optimize high-level code
5. **Historical Significance**: x86's long history illustrates computer architecture trends
